Control Dimensions

User Control (Demand-Side)

How to steer recommendations based on what the user wants.

Explicit Intent

Natural language instructions → OxygenREC, InstructRec

Prompt templates → P5

Implicit Preference

Preference pairs (DPO) → DPO, RankGR

Sequential patterns → TIGER, GPR

Multi-Interest

Multi-head prediction → GPR, OneRanker

Beam diversity → PROMISE

Key Insight: OxygenREC's Contextual Reasoning Instructions bridge natural language to generative backbone.

Item Control (Supply-Side)

How to steer recommendations based on item properties.

Semantic Similarity

Hierarchical Semantic IDs → TIGER, GPR

Context-aware tokenization → ActionPiece

Attribute Match

Constrained decoding → GRank

Gap: Explicit attribute control unexplored

Key Insight: Semantic ID hierarchy enables coarse-to-fine control.

Business Control (Platform-Side)

How to steer recommendations for platform objectives.

Value Optimization

Value-aware fine-tuning → GPR

Task decoupling → OneRanker

Hierarchical DPO → RankGR

Diversity/Fairness

Beam diversity → PROMISE

Gap: Fairness in GR is underexplored

Key Insight: OneRanker separates interest from value optimization via task tokens.

Interaction Control (Interface-Side)

How control is applied and communicated.

Control Timing

Pre-generation → OxygenREC, P5

Mid-generation (PRM) → PROMISE

Post-generation → RankGR, GRank

Explainability

Prompt-based explanation → PEPLER

Gap: Explaining Semantic ID paths unexplored

Key Insight: PROMISE applies control at every step via Process Reward Model.

Control Mechanisms

Input

Conditioning

Prepending tokens, instructions, or embeddings to steer generation.

  • Control Tokens — OxygenREC, OneRanker
  • Natural Language — InstructRec, P5
  • Embedding Injection — CoLLM, LLaRA
Encoding

Latent Manipulation

Editing discrete codes or latent representations.

  • Hierarchical Editing — TIGER, GPR
  • Code Composition — RQ-VAE
  • Context-Aware Tokenization — ActionPiece
Training

Preference Optimization

Aligning generation with preferences via DPO/RLHF.

  • DPO — DPO, RankGR
  • RLHF / HEPO — GPR
  • Value-Aware Training — GPR
Generation

Constrained Decoding

Steering output via PRM guidance during inference.

  • Process Reward Model — PROMISE
  • Guided Beam Search — PROMISE
  • Refined Scoring — RankGR, GRank
Architecture

Multi-Objective Control

Balancing conflicting objectives through task tokens, separate heads, or Pareto optimization.

Task Tokens — OneRanker Multi-Head — GPR Decoupled Training — OneRanker
Interest vs Value Diversity vs Relevance Short-term vs Long-term
