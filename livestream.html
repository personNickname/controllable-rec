Entertainment livestream has 4 unique properties that make controllability both harder and more important than static-item recommendation.

Gift value is unique to livestream — a real-time, voluntary revenue signal reflecting entertainment quality, not purchase intent.

Axis 1: Control Targets

What entity are we steering? Mapped from static-item to livestream context.

User Control (Demand-Side)

Gift propensity, entertainment mood, watch duration intent.

Explicit Intent

"I want gaming streams" — genre/mood intent

Implicit Preference

Gift history (strongest signal), watch duration, comment sentiment

Context State

Late-night vs daytime content, social context

Multi-Interest

Gaming + singing + chat — multiple entertainment genres

Key: Gift history provides a much stronger preference signal than clicks. A user who gifts 100元 to a singer has expressed far more intent than one who clicked.

Item Control (Supply-Side)

Dynamic stream = sequence of 30s segments with real-time changing attributes.

Content Type

Stream genre: gaming / singing / chat / outdoor

Attributes

Streamer style, energy level, interaction intensity, audience size

Semantic Similarity

Foresight Semantic ID: 30s segment → VQ-VAE → [genre, style, moment_type]

Quality Signal

Real-time: comments/min, gifts/min, viewer count trend

Key: A livestream is not one item — it's a sequence of evolving micro-items. Controlling "when to recommend" (during high-light) is where real value lies.

Business Control (Platform-Side)

Gift value, consume value, and streamer ecosystem health.

Value Optimization

Gift value maximization: α×consume_value + β×gift_value

Strategic Goals

New streamer cold-start, genre diversity, ecosystem health

Diversity / Fairness

Prevent winner-take-all — streamers who get no exposure will quit

Compliance

Real-time content moderation, age-appropriate gating

Key: Streamer ecosystem is more fragile than product catalogs. A streamer who gets no exposure will quit (unlike a product that stays in inventory).

Axis 2: Control Interfaces

How is control delivered? Real-time content creates fundamentally different interface requirements.

Control Timing

When in the pipeline control is applied — mid-generation is unique to livestream.

Pre-Generation

User opens feed → rank streams by predicted consume value and gift propensity

Mid-Generation

User watching → decide when to surface next stream. Interrupting during high-light hurts gift value; waiting during lull loses the user

Post-Generation

Rerank by real-time state — is streamer still live? Current energy? Recent gift surge?

Key: Mid-generation timing is unique to livestream. The system must decide WHEN to intervene, not just WHAT to recommend.

Control Granularity

How fine-grained — from genre to 30-second segments.

Coarse

Stream type: gaming / singing / chat / outdoor

Medium

Streamer style + current content theme

Fine

This 30s window is a high-light → boost now. Maps to Foresight Semantic ID

Key: Foresight's 30s segment Semantic ID enables fine-grained control — distinguish "this streamer right now" from "5 minutes ago."

Control Explainability

Why this recommendation — temporal explanations become possible.

Prompt-Based

"This streamer matches your interest in gaming"

Moment-Based

"This streamer is in a high-energy moment right now" — real-time state as explanation

Social Proof

"500 viewers are gifting right now" — crowd signal as explanation

Key: Livestream enables temporal explanations — "why now" is as important as "why this streamer."

Control Feedback

Continuous real-time signals instead of post-hoc ratings.

Implicit Real-time

Watch duration, comments, gifts, follows, screen-off — continuous during consumption

Gift as Feedback

Strongest positive signal. Amount provides continuous preference value, not binary

Leave as Feedback

When they leave matters — leaving during lull vs high-light mean different things

Key: A gift mid-stream is both a business metric AND a feedback signal. This creates a closed loop: feedback → update recommendation of the SAME stream to other users.

Axis 3: Control Mechanisms

How is control implemented? Each mechanism maps to livestream-specific signals and techniques.

Input

Conditioning

Segment Semantic ID, streamer profile tokens, real-time state embedding as inputs.

  • Segment Semantic ID — Foresight VQ-VAE codes for 30s snapshots
  • Streamer Profile — Genre, style, historical gift rate
  • Real-time State — Energy, interaction intensity, viewer count
  • Context Tokens — Time-of-day, user mood
Training

Preference Optimization

DPO on gift/consume value preference pairs with natural hierarchy.

  • Gift-based DPO — Gifted stream > non-gifted
  • Hierarchical DPO — 大额 > 小额 > 长看 > 点击 > 跳过
  • Consume Value Opt — High engagement > low engagement
  • Joint Training — α×L_gift + β×L_consume
Generation

Constrained Decoding

High-light moment detection as real-time PRM, stream-state-aware beam search.

  • High-light PRM — LiveForesighter moment detection
  • State-aware Beam — Weighted by live/energy/interaction
  • Segment Constraints — Fix top codes for genre control
  • Crowd Signals — Gift/comment rate as beam score
Post-generation

Post-Processing

Multi-signal reranking with real-time state refresh at high frequency.

  • Multi-signal Reranking — relevance + gift + consume + fairness
  • State Refresh — Drop ended, boost high-light, penalize lull
  • Streamer Diversity — MMR across genres
  • Fairness Bonus — Exposure equity for new streamers

Research Gaps

5 gaps unique to entertainment livestream, identified through the 3-axis taxonomy mapping.

P0

Temporal Semantic ID Evolution

No work models transitions between segments. Predicting "high-light in 2 minutes" enables proactive recommendation — boosting consume value by timing.

Item × Granularity × Conditioning
Medium Difficulty High Consume Value Impact
P0

Gift Value Preference Hierarchy

Gift amounts have natural hierarchy (大额 > 小额 > 评论 > 点击 > 跳过). No work applies Listwise DPO to this continuous signal.

Business × Preference Optimization
Medium Difficulty High Gift Value Impact
P1

Real-time Feedback as Decoding Signal

Gift and comment signals arrive continuously but only used for post-processing. Feeding them into decoding could leverage social proof and gifting contagion.

User × Feedback × Constrained Decoding
High Difficulty High Consume + Gift Impact
P1

Cross-Stream Session Control

Users hop between streams. Interrupting during high-light hurts gift value; waiting during lull loses the user. No work optimizes transition timing.

User × Timing × Conditioning
High Difficulty High Consume Value Impact
P2

Streamer Cold-Start Fairness

New streamers have no gift history and no Semantic ID history. The "item" doesn't exist until the stream starts.

Business × Post-Processing × Pref Opt
Medium Difficulty Medium Gift Value Impact
